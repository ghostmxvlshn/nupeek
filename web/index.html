From package blind spots to behavior-accurate decisions

Nupeek is designed for the gap between package APIs and package reality.

The problem

Agents can call NuGet APIs long before they can inspect the implementation.

That often means generic advice, fragile fixes, and incomplete debugging when the real behavior lives inside a compiled dependency.

  • Why is ServiceBusSender failing during retries?
  • Is this Polly policy safe for this path?
  • What changed between package versions?
The solution

Pull only the source your agent needs and make it inspectable immediately.

Nupeek keeps the workflow narrow, local, and deterministic so the model can reason from real code instead of inferred package behavior.

  • Find package and version
  • Select the best runtime target and TFM
  • Locate the requested type in the correct assembly
  • Generate readable C# plus quick lookup files
Specific types

Decompile the exact class your agent is reasoning about, not the entire package.

Local-first

Write outputs to your workspace so debugging context survives across prompts and sessions.

Agent-ready

Built for terminal-native workflows where speed and implementation accuracy matter.

Install and run

Get Nupeek into the workflow fast.

Use the .NET global tool or the one-line installer, then point Nupeek at the package type your agent needs to understand.

.NET global tool

Standard setup for local developer machines.

dotnet tool install -g Nupeek
nupeek --help
One-line installer

Quick bootstrap for terminals and ephemeral environments.

curl -fsSL \
  https://raw.githubusercontent.com/ghostmxvlshn/nupeek/main/install.sh | bash

How it works

A narrow pipeline built for package-level clarity.

Nupeek avoids broad decompilation passes and focuses on the exact code path your agent is trying to reason about.

01

Resolve the package

Read the package identity and version the project is actually using.

02

Choose the runtime target

Pick the most relevant target framework and assembly for the type lookup.

03

Extract the requested type

Find the symbol and generate readable decompiled C# where the agent can inspect it.

04

Write stable local artifacts

Save source and manifests into deps-src/ for repeatable follow-up analysis.
