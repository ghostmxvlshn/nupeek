FOR COPILOT, CLAUDE CODE, CODEX, AND OTHER CLI AGENTS

Give your AI agent the source it’s missing.

Nupeek decompiles only the NuGet types you actually use into a local deps-src/ folder — so your agent can reason from real implementation, not guesses.

Install Nupeek See workflow 
$ nupeek type --package Azure.Messaging.ServiceBus \
  --type Azure.Messaging.ServiceBus.ServiceBusSender \
  --out deps-src --dry-run false

✓ outputPath: deps-src/packages/azure.messaging.servicebus/.../ServiceBusSender.decompiled.cs
✓ indexPath: deps-src/index.json
✓ manifestPath: deps-src/manifest.json

The problem

Agents can call package APIs, but they often can’t see implementation internals quickly. Hunting through docs or source repos slows down debugging, architecture review, and rule-writing.

The Nupeek approach

Resolve package → pick TFM → locate type assembly → decompile one type → write index + manifest. Fast, deterministic, local-first.

Install

.NET global tool (recommended)

dotnet tool install -g Nupeek
nupeek --help

One-line installer

curl -fsSL https://raw.githubusercontent.com/ghostmxvlshn/nupeek/main/install.sh | bash

Workflow

  1. Pick target symbol
    Identify the package type your code depends on.
  2. Run Nupeek
    Generate decompiled source for that type only.
  3. Let agent inspect deps-src
    Your coding assistant now has real implementation context.

Open source and local-first

Nupeek is MIT licensed. No hosted dependency, no cloud lock-in — just files you control.
